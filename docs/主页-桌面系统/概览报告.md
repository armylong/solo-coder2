# 主页-桌面系统概览报告

## 一、系统概述

### 1.1 系统定位

主页-桌面系统是一个模仿 macOS 风格的 Web 桌面操作系统，为用户提供一个直观、易用的应用管理和交互界面。该系统作为整个阿米龙平台的入口，整合了各类应用程序，提供统一的用户体验。

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端语言 | Go 语言 |
| Web 框架 | Gin |
| 数据库 | SQLite |
| 前端 | 原生 JavaScript + CSS3 |
| 样式风格 | macOS 风格设计 |

### 1.3 核心功能模块

- **用户认证系统**：登录、注册、登出
- **应用管理**：应用商店、应用安装/卸载
- **桌面系统**：图标管理、拖拽排序、位置记忆
- **Dock 栏**：快速启动、应用状态指示、位置可调
- **窗口系统**：多窗口管理、拖拽、缩放、最小化/最大化/全屏
- **权限控制**：基于角色的应用访问权限

### 2.2 核心数据模型

#### 2.2.1 应用表 (tb_app)

| 字段 | 类型 | 说明 |
|------|------|------|
| app_id | INTEGER | 主键，自增 |
| app_name | TEXT | 应用名称，唯一 |
| desc | TEXT | 应用描述 |
| icon | TEXT | 应用图标（emoji） |
| url | TEXT | 应用入口 URL |
| type | INTEGER | 应用类型（1=应用，2=游戏） |
| permission | INTEGER | 权限级别 |
| status | INTEGER | 状态（1=启用） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 2.2.2 桌面系统应用表 (tb_desktop_os_app)

系统默认应用，包含各类游戏和工具应用。

#### 2.2.3 用户应用表 (tb_user_app)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| uid | INTEGER | 用户 ID |
| app_id | INTEGER | 应用 ID |
| ext | TEXT | 扩展信息（JSON 格式，包含位置等） |
| status | INTEGER | 状态（1=已安装，2=已卸载） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**扩展信息 (UserAppExt) 结构：**
```json
{
  "position": "desktop",  // 位置：desktop 或 dock
  "x": 2,                  // 桌面 X 坐标百分比（相对于桌面宽度的百分比，0-100）
  "y": 2,                  // 桌面 Y 坐标百分比（相对于桌面高度的百分比，0-100）
  "dock_index": 0          // Dock 栏索引
}
```

## 三、主要功能特性

### 3.1 桌面系统

- **图标管理**：应用图标可放置在桌面任意位置
- **拖拽功能**：支持图标拖拽移动位置
- **位置记忆**：用户的图标位置会被持久化保存
- **右键菜单**：支持查看应用信息、卸载应用

### 3.2 Dock 栏

- **快速启动**：固定常用应用到 Dock 栏
- **状态指示**：运行中的应用显示指示器
- **位置可调**：支持拖拽到屏幕四边（上/下/左/右）
- **拖拽排序**：Dock 栏内图标可拖拽调整顺序

### 3.3 窗口系统

- **多窗口支持**：可同时打开多个应用窗口
- **拖拽移动**：窗口标题栏可拖拽移动
- **边缘缩放**：窗口边缘支持缩放调整大小
- **状态管理**：支持最小化、最大化、全屏模式
- **层级管理**：支持窗口前后层级切换

### 3.4 用户系统

- **登录/注册**：支持用户账号注册和登录
- **权限分级**：
  - 普通用户（permission=0）：只能看到权限为 0 的应用
  - 管理员（permission=2）：可看到权限为 0 和 2 的应用
  - 超级管理员（permission=1）：可看到所有应用
- **令牌管理**：使用 Authorization 头进行身份验证

### 3.5 应用商店

- **自动安装**：Long Store 应用会自动为新用户安装
- **权限过滤**：根据用户权限显示对应可见应用
- **安装/卸载**：支持从应用商店安装和卸载应用

## 四、界面设计

### 4.1 整体风格

系统采用模仿 macOS 的设计风格，包含以下特色：

- **顶部状态栏**：显示 Apple 标志、系统名称、时间
- **Apple 菜单**：点击 Apple 标志显示系统菜单
- **毛玻璃效果**：使用 CSS `backdrop-filter: blur()` 实现
- **渐变背景**：桌面使用紫色渐变背景

### 4.2 关键组件

| 组件 | 说明 |
|------|------|
| 状态栏 (StatusBar) | 高度 25px，显示系统信息和时间 |
| Apple 菜单 | 包含关于、用户信息、系统设置、退出登录等 |
| 桌面 (Desktop) | 应用图标容器，支持拖拽放置 |
| Dock 栏 | 底部/顶部/左侧/右侧应用快捷启动栏 |
| 窗口 (Window) | 应用运行容器，支持拖拽、缩放、状态切换 |

## 五、快速启动流程

1. **用户访问主页** → 加载桌面系统前端
2. **检查登录状态** → 从 localStorage 读取 auth_token
3. **加载桌面数据** → 调用 `/index/desktopOs` 接口
4. **渲染界面** → 根据返回数据渲染桌面和 Dock
5. **应用交互** → 用户可打开应用、拖拽图标等

## 六、系统特点总结

### 优势
- 模仿 macOS 的直观用户界面
- 完整的应用生命周期管理
- 灵活的窗口系统
- 基于角色的权限控制
- 轻量级技术栈，易于部署

### 适用场景
- 作为内部工具平台的统一入口
- 游戏中心和娱乐平台
- 个人 Web 桌面系统
