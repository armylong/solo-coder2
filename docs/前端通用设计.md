## 核心设计原则

### 1. `_index/` 资源隔离

每个页面目录下都有一个 `_index/` 子目录，存放该页面专属的 css、js、assets：

- **为什么用 `_index/`**：避免根目录下 html、css、js、子页面目录混在一起，保持根目录只有 `index.html` 和子页面目录
- **为什么叫 `_index/`**：下划线开头排在最前面，一眼就能找到资源目录

### 2. ES Modules

所有自己写的 JS 都是 ES Module：

- HTML 中统一使用 `<script type="module" src="_index/js/main.js">`
- 公共模块通过 `import` 引入，不再使用全局变量
- CDN 引入的外部库（如 Three.js）通过 `<script>` 先加载，模块内直接访问 `window.THREE`

### 3. 公共样式 + 页面样式

- `common/style.css`：全局 reset、CSS 变量、loading/toast/modal/buttons 等通用组件样式
- `_index/css/style.css`：页面专属样式，只写该页面特有的样式

### 4. 公共方法不重复实现

`common/ui.js` 提供了 `showLoading`、`hideLoading`、`showToast`、`escapeHtml`，各页面直接 import 使用，禁止自定义同名方法。

### JS 标准模板

```javascript
import { Auth, PERMISSION } from '/static/common/auth.js';
import { showLoading, hideLoading, showToast, escapeHtml } from '/static/common/ui.js';

const API_BASE = '/your_module';

document.addEventListener('DOMContentLoaded', () => {
    if (!Auth.requirePermission(PERMISSION.ADMIN)) {
        return;

    loadData();
});

async function fetchApi(action, params = {}) {
    return Auth.fetchApi(API_BASE, action, params);

async function loadData() {

        const result = await fetchApi('list', { page: 1 });
        renderList(result.data);

        showToast(error.message || '加载失败', 'error');
    } finally {
        hideLoading();
    }
}
```

### common/auth.js

认证 + API 调用 + 权限判断，所有页面必须引入。

#### 常量

| 常量 | 值 | 说明 |
|---|---|---|
| `PERMISSION.NORMAL` | 0 | 普通用户 |
| `PERMISSION.ADMIN` | 1 | 管理员 |
| `PERMISSION.SUPER_ADMIN` | 2 | 超级管理员 |
| `RESPONSE_CODE.UNAUTHORIZED` | -401 | 未登录 |
| `RESPONSE_CODE.FORBIDDEN` | -403 | 无权限 |

#### API

| 方法 | 说明 |
|---|---|
| `Auth.getToken()` / `setToken(token)` / `clearToken()` | Token 管理 |
| `Auth.isAuthenticated()` | 是否已登录 |
| `Auth.fetchWithAuth(url, options)` | 带 Token 的原生 fetch |
| `Auth.fetchApi(baseUrl, action, params)` | 统一 API 调用 |
| `Auth.getUserPermission()` | 从 JWT 解析权限 |
| `Auth.isSuperAdmin()` / `isAdmin()` | 权限判断 |
| `Auth.requirePermission(level)` | 前端权限拦截 |
| `Auth.decodeToken()` | 解析 JWT Payload |
| `Auth.getUser()` | 获取当前用户信息（异步，通过 SessionData） |

`Auth.fetchApi` 内部流程：

```
1. 拼接 URL: baseUrl + '/' + action
2. 自动带 Token (Authorization Header)
3. POST JSON 请求
4. 自动处理响应码：
   - -401 → 清 Token + 清 SessionData + 跳首页
   - -403 → 弹窗 + 返回上页
   - 其他负数 → throw Error
   - >= 0 → 返回 result
```

### common/session-data.js

会话数据缓存层，按需获取登录后才能访问的业务数据，缓存到 localStorage。

**设计原则**：谁用谁调，不用不调。各页面按需 fetch 自己需要的 keys，缓存有则直接返回，没有才请求后端。

#### API

| 方法 | 说明 |
|---|---|
| `SessionData.fetch(keys)` | 异步获取数据：先查缓存，缺失的 key 才请求后端，合并后返回 |
| `SessionData.get(keys)` | 同步获取数据：只从缓存读，不请求后端 |
| `SessionData.clear()` | 清除所有缓存（退出登录、被踢下线时调用） |
| `SessionData.refresh(keys)` | 清除指定 key 的缓存并重新请求 |

#### 使用方式

```javascript
import { SessionData } from '/static/_common/session-data.js';

// 异步获取（缓存没有会请求后端）
const data = await SessionData.fetch(['user', 'ppz_user']);
const user = data.user;
const ppzUser = data.ppz_user;

// 同步获取（只从缓存读，不请求后端）
const cached = SessionData.get(['user']);

// 刷新指定 key
await SessionData.refresh(['user']);

// 清除所有缓存
SessionData.clear();
```

#### 后端接口

`POST /session_data/sessionData`

请求：
```json
{ "keys": ["user", "ppz_user"] }
```

响应：
```json
{
  "code": 0,
  "data": {
    "user": { "uid": 1, "account": "test", "name": "测试", ... },
    "ppz_user": { "car_count": 1, "status": 1, "driver_status": 1, ... }
  }
}
```

#### 已注册的 key

| key | 数据来源 | 说明 |
|---|---|---|
| `user` | tb_user | 用户基础信息（不含密码） |
| `ppz_user` | tb_ppz_user | 拼拼坐用户扩展信息（car_count、driver_status 等） |

新增业务模块只需在后端 `business/session_data` 的 `dataFetchers` 中注册新的 key 和对应的 DataFetcher 函数即可。

#### 缓存生命周期

| 场景 | 处理 |
|---|---|
| 页面按需获取 | `fetch` 自动查缓存，缺失才请求后端 |
| 退出登录 | `SessionData.clear()` 清除所有缓存 |
| 被踢下线（401） | `Auth.fetchApi` 自动调 `SessionData.clear()` |
| 修改个人信息后 | `SessionData.refresh(['user'])` 刷新指定 key |

### common/login-modal.js + login-modal.css

登录弹窗组件，提供全局登录拦截能力。弹窗采用毛玻璃背景 + 圆角卡片设计。

**核心方法**：

| 方法 | 说明 |
|---|---|
| `requireLogin(callback)` | 登录守卫：已登录直接执行 callback，未登录弹登录弹窗，登录成功后自动执行 callback |

**使用场景**：

```javascript
import { requireLogin } from '/static/common/login-modal.js';

// 点击需要登录的功能
requireLogin(() => {
    window.location.href = '/static/ppz-m/profile/';
});

// 点击需要登录的豆腐块
requireLogin(() => {
    doSomething();
});
```

**弹窗特性**：
- 毛玻璃背景（backdrop-filter: blur）
- 登录/注册切换
- 登录成功自动执行回调并关闭弹窗
- 点击遮罩或关闭按钮可关闭

**HTML 要求**：需要引入 `login-modal.css`，弹窗 DOM 由 JS 动态创建。

## 前后端协作

### 核心原则

**前端统一带 Token，后端决定是否校验。**

前端不需要知道"这个接口要不要带 Token"，所有请求都带，后端不用就忽略。

### API 调用约定

前端统一使用 `Auth.fetchApi(baseUrl, action, params)` 调用后端接口：

```javascript
const result = await Auth.fetchApi('/auth', 'login', { account, password });
const result = await Auth.fetchApi('/user', 'info', {});
const result = await Auth.fetchApi('/user_management', 'userList', { page: 1 });
```

### 后端路由与中间件对应

| 路由组 | 中间件 | 前端处理 |
|---|---|---|
| 无中间件 | 不校验 Token | 正常调用，Token 有则带无则不带 |
| `Middleware` | 校验 Token | Token 无效返回 -401，Auth 自动处理 |
| `Middleware` + `RequireAdmin` | 校验 Token + 管理员权限 | 非管理员返回 -403，Auth 自动处理 |
| `Middleware` + `RequireSuperAdmin` | 校验 Token + 超管权限 | 非超管返回 -403，Auth 自动处理 |

### 响应格式

```json
{
    "code": 0,
    "message": "success",
    "data": { ... }
}
```

| code | 含义 | 前端处理 |
|---|---|---|
| 0 | 成功 | 返回 result |
| -401 | 未登录 | 清 Token + 跳首页 |
| -403 | 无权限 | 弹窗 + 返回上页 |
| 其他负数 | 业务错误 | throw Error |
| 正数 | 业务状态码 | 返回 result |

## 页面权限控制

### 前端权限拦截

```javascript
import { Auth, PERMISSION } from '/static/common/auth.js';

document.addEventListener('DOMContentLoaded', () => {
    if (!Auth.requirePermission(PERMISSION.ADMIN)) {
        return;
    }
    loadData();
});
```

### UI 元素权限控制

```javascript
if (Auth.isSuperAdmin()) {
    document.getElementById('addAppBtn').style.display = 'flex';
}
```

### 双重保障

- 前端拦截 → 避免无权限用户看到不该看的 UI
- 后端拦截 → 即使绕过前端，接口也会返回 -403
