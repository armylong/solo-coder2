# 后台 - 概览（ppz-admin/index.html）

## 产品定位

拼拼坐后台管理系统是面向运营管理人员的 PC 端应用，提供司机管理、车辆审核、运营区域管理等核心管理功能。管理员通过审核司机提交的车辆信息，控制平台司机准入；通过运营区域管理，配置拼车服务的覆盖区域。

### API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /ppz_admin/overviewStats | 获取概览统计 | 页面加载、点击刷新 |

**响应数据**：
```json
{
  "total_drivers": 128,
  "banned_drivers": 5,
  "total_cars": 256
}
```

### 数据统计逻辑

**司机总数**：从 tb_ppz_cars 表中获取所有不重复的 uid 数量

**封禁司机数**：遍历所有司机 uid，检查 tb_ppz_user 中 driver_status = 2 的数量

**车辆总数**：统计 tb_ppz_cars 表中 status = 1 的记录数

## 核心管理流程

### 车辆审核流程

```
司机提交车辆 → 进入待审核列表 → 管理员查看详情
                                    ↓
                          ┌─── 通过 → 创建车辆记录 → 司机认证成功
                          └─── 驳回 → 司机可编辑重新提交
```

### 司机封禁流程

```
发现违规司机 → 封禁（填写原因）→ 所有车辆审核置为驳回 → 司机无法操作
                                                          ↓
                                              解封 → 司机可重新提交审核
```

### 运营区域管理流程

```
运营区域列表 → 添加区域 → 围栏绘制子页面（悬浮）→ 提交保存 → 返回列表
             → 编辑区域 → 加载已有围栏 → 修改 → 二次确认 → 提交更新
             → 停用/启用/删除区域
```

## API 接口总览

| 接口 | 说明 |
|------|------|
| POST /ppz_admin/overviewStats | 概览统计数据 |
| POST /ppz_admin/driverList | 司机列表（分页） |
| POST /ppz_admin/banDriver | 封禁司机 |
| POST /ppz_admin/unbanDriver | 解封司机 |
| POST /ppz_admin/carAuditOverviewStats | 审核统计 |
| POST /ppz_admin/carAuditList | 审核列表（按司机聚合） |
| POST /ppz_admin/approveCarAudit | 审核通过 |
| POST /ppz_admin/rejectCarAudit | 审核驳回 |
| POST /business_area/list | 运营区域列表 |
| POST /business_area/get | 获取区域详情 |
| POST /business_area/create | 创建运营区域 |
| POST /business_area/update | 更新运营区域 |
| POST /business_area/updateStatus | 更新区域状态 |
| POST /business_area/delete | 删除运营区域 |

## 技术特点

- **原生实现**：无框架依赖，使用原生 HTML/CSS/JS (ES6 Module)
- **PC 端布局**：左侧固定侧边栏 + 右侧内容区
- **地图集成**：高德地图 JSAPI + Clipper.js 布尔运算
- **数据聚合**：审核列表按司机维度聚合，提升审核效率
- **侧边栏导航**：所有页面共享侧边栏，当前页面高亮
