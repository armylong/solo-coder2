# 后台 - 车辆审核（ppz-admin/car-audit/）

## 一、页面概述

车辆审核页面是管理员审核司机提交的车辆信息的核心页面。支持按审核状态查看、按用户信息筛选、审核通过/驳回等操作。审核列表按司机聚合展示。

**访问路径**：`/static/ppz-admin/car-audit/`

**前置条件**：管理员已登录

## 三、功能模块

### 3.1 统计卡片

| 统计项 | 图标 | 颜色 | 说明 |
|--------|------|------|------|
| 待审核 | ⏳ | 黄色 | audit_status=1 的记录数 |
| 已通过 | ✅ | 绿色 | audit_status=2 的记录数 |
| 已驳回 | ❌ | 红色 | audit_status=3 的记录数 |

### 3.2 Tab 切换

| Tab | audit_status | 说明 |
|-----|-------------|------|
| 待审核 | 1 | 默认选中，优先处理 |
| 已驳回 | 3 | 查看被驳回的记录 |
| 已通过 | 2 | 查看已通过的记录 |

每个 Tab 旁显示对应数量。

### 3.3 筛选条件

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 用户ID | number | 精确匹配 |
| 账号 | text | 模糊匹配 |
| 姓名 | text | 模糊匹配 |
| 手机号 | text | 模糊匹配 |

### 3.4 审核列表（按司机聚合）

列表按司机维度聚合，每个司机卡片包含：
- 司机基本信息（UID、账号、姓名、手机号）
- 各状态车辆数量统计
- 该司机当前 Tab 状态下的车辆列表

**车辆卡片展示**：
| 信息 | 说明 |
|------|------|
| 车辆型号 | audit_data.car_model |
| 座位数 | audit_data.seats |
| 审核状态 | audit_status |
| 操作按钮 | 查看详情、通过、驳回 |

### 3.5 审核通过

**流程**：
```
点击「通过」→ 调用 approveCarAudit API
  → 成功 → 刷新列表
  → 失败 → 显示错误信息
```

**后端处理**：
1. 获取审核记录
2. 如果 car_id=0（新车辆）→ 创建车辆记录
3. 如果 car_id>0（编辑车辆）→ 更新车辆记录
4. 回填 car_id 到审核记录
5. 更新 audit_status=2
6. 写入审核日志
7. 递增用户 car_count

### 3.6 审核驳回

**流程**：
```
点击「驳回」→ 弹出驳回原因输入框
→ 填写驳回原因 → 调用 rejectCarAudit API
  → 成功 → 刷新列表
  → 失败 → 显示错误信息
```

**后端处理**：
1. 更新 audit_status=3
2. 记录驳回原因
3. 写入审核日志

### 3.7 查看详情

**流程**：
```
点击「查看详情」→ 弹出审核详情弹窗
→ 展示完整车辆信息和审核信息
→ 可直接在弹窗中操作通过/驳回
```

### 3.8 分页

- 默认每页 20 条（按司机数量分页）
- 支持上一页/下一页
- 显示总记录数

## 四、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /ppz_admin/carAuditOverviewStats | 审核统计 | 页面加载 |
| POST /ppz_admin/carAuditList | 审核列表 | 页面加载、切换Tab、筛选、翻页 |
| POST /ppz_admin/approveCarAudit | 审核通过 | 点击通过 |
| POST /ppz_admin/rejectCarAudit | 审核驳回 | 点击驳回确认 |
| POST /ppz_admin/getCarDetail | 车辆详情 | 点击查看详情 |

**请求参数**（carAuditList）：
```json
{
  "page": 1,
  "page_size": 20,
  "audit_status": 1,
  "uid": 0,
  "account": "",
  "name": "",
  "phone": ""
}
```

## 五、数据查询逻辑

### 5.1 审核列表查询

```
1. 从 tb_ppz_car_audit 获取指定状态的不重复 uid
2. 按 uid、account、name、phone 筛选
3. 分页截取
4. 对每个 uid：
   a. 获取 tb_user 信息
   b. 获取该 uid 在当前审核状态下的所有审核记录
   c. 统计各状态数量
   d. 组装车辆列表
```

### 5.2 聚合展示

审核列表按司机维度聚合，而非按单条审核记录展示。这样管理员可以：
- 一次性审核同一司机的多辆车辆
- 了解每个司机的整体审核情况
- 避免同一司机多次出现在列表中
