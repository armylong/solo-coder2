# 后台 - 运营区域管理（ppz-admin/business-area/）

## 一、页面概述

运营区域管理是拼拼坐后台的核心功能之一，用于管理拼车服务的覆盖区域。包含区域列表页和围栏绘制子页面两部分。

**区域列表页**：展示所有运营区域，支持状态筛选、名称搜索、启用/停用/删除操作。

**围栏绘制子页面**：基于高德地图的围栏绘制工具，通过选择行政区划或手动绘制的方式定义服务区域，支持围栏的合并（并集）和裁剪（差集）操作。子页面以悬浮方式覆盖在列表页上方，不覆盖侧边栏。

**访问路径**：`/static/ppz-admin/business-area/`

**前置条件**：管理员已登录

## 三、核心概念

### 3.1 双层围栏设计

| 围栏类型 | 颜色 | 含义 | 数据结构 |
|---------|------|------|---------|
| 蓝色围栏 | 蓝色 | 待确认的围栏区域，作为操作素材 | 2D数组：`[path1, path2, ...]`，每个 path 是 `[[lng, lat], ...]` |
| 绿色围栏 | 绿色 | 已确认的服务区域，支持含孔洞多边形 | 3D数组：`[polygon1, polygon2, ...]`，每个 polygon 是 `[outerRing, hole1, hole2, ...]` |

**数据结构对比**：
```
蓝色围栏（2D）：
[
  [[lng,lat], [lng,lat], ...],  // path1
  [[lng,lat], [lng,lat], ...],  // path2
]

绿色围栏（3D，支持孔洞）：
[
  [                              // polygon1
    [[lng,lat], [lng,lat], ...], // 外环
    [[lng,lat], [lng,lat], ...], // 孔洞1
  ],
  [                              // polygon2
    [[lng,lat], [lng,lat], ...], // 外环（无孔洞）
  ],
]
```

### 3.2 布尔运算

使用 Clipper.js 库进行多边形布尔运算：

| 操作 | 运算类型 | 效果 | 说明 |
|------|---------|------|------|
| 加入围栏 | 并集（Union） | 绿色 ∪ 蓝色 | 将蓝色区域合并到绿色区域 |
| 删除围栏 | 差集（Difference） | 绿色 - 蓝色 | 从绿色区域中减去蓝色区域 |

**运算流程**：
```
1. 将绿色围栏和蓝色围栏转换为 Clipper.js 的 IntPoint 格式
2. 使用 Clipper.Execute() 执行布尔运算
3. 从 PolyTree 结果中提取多边形（含孔洞）
4. 转换回经纬度坐标格式
```

## 四、功能模块

### 4.1 区域列表

**列表字段**：

| 字段 | 说明 |
|------|------|
| 区域ID | 自增主键 |
| 区域名称 | 区域的名称 |
| 围栏数据 | 点击"查看围栏"显示美化后的JSON |
| 状态 | 正常（1）/ 停用（2） |
| 更新时间 | 最后修改时间 |
| 操作 | 查看/编辑、停用/启用、删除 |

**筛选功能**：
- 状态筛选：全部、正常、停用
- 名称搜索：按区域名称模糊搜索

### 4.2 添加区域

**流程**：
```
1. 点击"添加区域"按钮
2. 打开围栏绘制子页面（area_id=0）
3. 在子页面绘制围栏
4. 点击"提交保存"
5. 弹出区域名称输入框（必填）
6. 确认后提交到后端接口
7. 保存成功后关闭子页面，刷新列表
```

### 4.3 编辑区域

**流程**：
```
1. 点击"查看/编辑"按钮
2. 打开围栏绘制子页面（area_id>0）
3. 自动加载已有围栏数据
4. 修改围栏
5. 点击"提交保存"
6. 弹出二次确认框（是否确认更改）
7. 确认后提交到后端接口
8. 保存成功后关闭子页面，刷新列表
```

**注意**：编辑时不弹备注输入框，只弹二次确认

### 4.4 停用/启用区域

**流程**：
```
点击"停用" → 二次确认 → 调用 updateStatus 接口 → 刷新列表
点击"启用" → 二次确认 → 调用 updateStatus 接口 → 刷新列表
```

### 4.5 删除区域

**流程**：
```
点击"删除" → 二次确认（显示区域名称）→ 调用 delete 接口 → 刷新列表
```

### 4.6 查看围栏数据

**流程**：
```
点击"查看围栏" → 弹出模态框 → 显示美化后的JSON数据
```

## 五、围栏绘制子页面

### 5.1 页面特性

- 无侧边栏，全屏地图展示
- 悬浮在列表页上方，不覆盖菜单栏
- 上下左右有阴影效果
- 通过 `area_id` URL 参数区分新建/编辑模式

### 5.2 行政区划选择

**三级联动**：省份 → 城市 → 区县

**流程**：
```
选择省份 → 加载城市列表 → 选择城市 → 加载区县列表 → 选择区县
→ 自动搜索该区县 → 获取边界数据 → 绘制蓝色围栏
```

### 5.3 搜索行政区域

**流程**：
```
输入关键词（如：朝阳区）→ 调用高德搜索API
→ 获取匹配的行政区划 → 获取边界数据 → 绘制蓝色围栏
```

### 5.4 加入围栏（并集操作）

```
1. 检查蓝色围栏是否有效（至少3个点）
2. 将蓝色围栏转换为绿色格式（2D → 3D）
3. 如果绿色围栏为空 → 直接赋值
4. 如果绿色围栏不为空 → 执行并集运算
5. 重新渲染绿色围栏
6. 清除蓝色围栏
```

### 5.5 删除围栏（差集操作）

```
1. 检查蓝色围栏和绿色围栏是否有效
2. 将蓝色围栏转换为绿色格式
3. 执行差集运算（绿色 - 蓝色）
4. 重新渲染绿色围栏
5. 清除蓝色围栏
```

### 5.6 提交保存

**新建模式**：
```
1. 检查绿色围栏是否为空
2. 弹出区域名称输入框（必填）
3. 调用 create 接口提交数据
4. 成功后通过 postMessage 通知列表页
```

**编辑模式**：
```
1. 检查绿色围栏是否为空
2. 弹出二次确认框
3. 确认后调用 update 接口提交数据
4. 成功后通过 postMessage 通知列表页
```

## 六、技术实现

### 6.1 Clipper.js 集成

**坐标转换**：
- 高德地图使用经纬度（浮点数）
- Clipper.js 使用整数坐标
- 转换倍率：1e6（保留6位小数精度）

```javascript
function toClipperPath(ring) {
    return ring.map(p => ({ X: Math.round(p[0] * 1e6), Y: Math.round(p[1] * 1e6) }));
}

function fromClipperPath(path) {
    return path.map(p => [p.X / 1e6, p.Y / 1e6]);
}
```

### 6.2 子页面通信

列表页与子页面通过 iframe + postMessage 通信：

```javascript
// 列表页：打开子页面
function openFencePage(areaId) {
    iframe.src = 'fence/?area_id=' + areaId;
    overlay.classList.add('show');

    window.addEventListener('message', function handler(e) {
        if (e.data.type === 'fenceSaved') {
            overlay.classList.remove('show');
            refreshList();
        } else if (e.data.type === 'fenceCancel') {
            overlay.classList.remove('show');
        }
    });
}

// 子页面：保存成功后通知列表页
window.parent.postMessage({ type: 'fenceSaved' }, '*');
```

### 6.3 凹多边形绘制

蓝色围栏绘制时，需要设置 `bubble: true` 属性，使多边形的填充区域不拦截点击事件，从而支持在已有围栏内部继续绘制凹形区域。

## 十、设计说明

1. **双层围栏**：蓝色作为操作素材，绿色作为最终结果，分离操作和结果
2. **布尔运算**：使用 Clipper.js 确保多边形运算的准确性和稳定性
3. **孔洞支持**：绿色围栏支持含孔洞的多边形，适用于剔除围栏内部区域
4. **坐标简化**：超过100个点的路径自动简化，保证渲染性能
5. **悬浮子页面**：围栏绘制页面以 iframe 悬浮方式展示，不覆盖侧边栏
6. **后端存储**：围栏数据通过 API 提交到后端保存，不再仅打印到控制台
7. **编辑确认**：编辑时弹出二次确认框，防止误操作
