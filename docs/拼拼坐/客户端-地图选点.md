# 用户端 - 地图选点（ppz-m/address-map/）

## 一、页面概述

地图选点页面基于高德地图 JSAPI 实现，提供地图浏览、地址搜索、选点确认等功能。用于添加或编辑常用地址时选择具体位置。

**访问路径**：
- 新增出发地：`/static/ppz-m/address-map/index.html?type=1`
- 新增目的地：`/static/ppz-m/address-map/index.html?type=2`
- 编辑地址：`/static/ppz-m/address-map/index.html?address_id=xxx`

**前置条件**：用户已登录

## 三、功能模块

### 3.1 地图浏览

- 全屏展示高德地图
- 支持拖拽、缩放操作
- 地图中央固定一个标记点（📍），随地图移动
- 地图移动结束后，自动获取中心点地址信息（逆地理编码）

### 3.2 地址搜索

**搜索流程**：
```
输入关键词 → 调用高德搜索API → 展示搜索结果列表
→ 点击搜索结果 → 地图定位到该位置 → 更新地址信息面板
```

**搜索功能**：
- 输入框实时搜索
- 支持清除搜索内容
- 搜索结果面板可折叠/展开
- 点击搜索结果自动定位

### 3.3 地址信息面板

- 固定在地图底部
- 实时显示当前中心点的地址信息
- 包含「选择此地址」按钮

**信息展示**：
| 信息 | 来源 | 说明 |
|------|------|------|
| 地址名称 | 逆地理编码 | 简短地址描述 |
| 地址详情 | 逆地理编码 | 完整地址信息 |

### 3.4 确认地址

**流程**：
```
点击「选择此地址」→ 弹出备注弹窗
→ 填写地址名称（如：我家、公司）
→ 地址详情自动填充（只读）
→ 点击确定 → 调用保存API → 返回常用地址页面
```

### 3.5 编辑模式

当 URL 中携带 `address_id` 参数时，进入编辑模式：

1. 页面加载时获取地址详情
2. 地图定位到该地址位置
3. 地址信息面板显示已有信息
4. 保存时调用更新接口（address_id > 0）

## 四、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /ppz/ppz_map/uploadAddress | 保存地址 | 点击备注弹窗确定 |
| POST /ppz/ppz_map/getAddressDetail | 获取地址详情 | 编辑模式页面加载 |
| 高德搜索API | 地址搜索 | 输入搜索关键词 |
| 高德逆地理编码API | 获取地址信息 | 地图移动结束 |

**请求参数**（uploadAddress - 新增）：
```json
{
  "map_type": 1,
  "remark": "我家",
  "gaode_data": {
    "name": "XX小区",
    "address": "XX市XX区XX路",
    "location": "116.397428,39.90923",
    ...
  }
}
```

**请求参数**（uploadAddress - 编辑）：
```json
{
  "address_id": 1,
  "map_type": 1,
  "remark": "我家（更新）",
  "gaode_data": { ... }
}
```

## 五、交互说明

1. **中心点标记**：标记点固定在地图中央，不随地图移动，代表当前选中的位置
2. **实时逆地理编码**：每次地图移动停止后，自动获取中心点的地址信息
3. **搜索结果定位**：点击搜索结果后，地图平滑移动到目标位置
4. **备注必填**：地址名称为必填项，不能为空
5. **返回确认**：如果已选择地址但未保存，返回时可能丢失数据
