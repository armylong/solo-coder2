# 用户端 - 底部导航组件（common/tab-nav）

## 一、组件概述

底部导航组件是全局公共组件，提供 Tab 式页面切换能力。每个 Tab 对应一个独立的 HTML 页面，点击 Tab 时通过浏览器原生页面跳转实现切换。

**设计理念**：每个 Tab 是独立页面，而非单页内的显示/隐藏切换。这样每个页面有独立的 URL、独立的加载生命周期，浏览器返回键天然可用。

## 三、API

### 3.1 导出方法

| 方法 | 参数 | 说明 |
|------|------|------|
| `renderTabNav(activeTab)` | activeTab: string，当前 Tab 的 id | 渲染底部导航栏，高亮当前 Tab |

### 3.2 使用示例

```javascript
import { renderTabNav } from '/static/common/tab-nav.js';

// 拼车页
renderTabNav('ride');

// 我的页
renderTabNav('profile');
```

## 五、交互行为

### 5.1 点击 Tab

| 场景 | 行为 |
|------|------|
| 点击当前激活的 Tab | 无操作（已在当前页面） |
| 点击不需要登录的 Tab | 直接跳转对应页面 |
| 点击需要登录的 Tab（已登录） | 直接跳转对应页面 |
| 点击需要登录的 Tab（未登录） | 拦截跳转，弹出登录弹窗，登录成功后跳转 |

### 5.2 页面跳转

使用 `<a>` 标签的原生 href 跳转，而非 JS 的 `location.href`：

- 浏览器返回键天然可用
- 页面有完整的加载生命周期
- URL 栏反映当前页面

### 5.3 登录拦截

需要登录的 Tab（`needAuth: true`）通过 `requireLogin` 实现拦截：

```javascript
// tab-nav.js 内部逻辑
item.addEventListener('click', function(e) {
    if (tab && tab.needAuth) {
        e.preventDefault();  // 阻止默认跳转
        requireLogin(function() {
            window.location.href = tab.href;  // 登录成功后跳转
        });
    }
});
```

## 七、集成方式

### 7.1 HTML

在页面中添加导航容器和 CSS 引入：

```html
<head>
    <link rel="stylesheet" href="/static/common/tab-nav.css">
</head>
<body>
    <div class="app-container">
        <main class="main-content">
            <!-- 页面内容，需设置 padding-bottom: 70px 避免被导航栏遮挡 -->
        </main>

        <nav class="tab-nav" id="tabNav"></nav>
    </div>
</body>
```

### 7.2 JS

```javascript
import { renderTabNav } from '/static/common/tab-nav.js';

document.addEventListener('DOMContentLoaded', function() {
    renderTabNav('ride');
});
```

## 八、注意事项

1. **HTML 容器必须有 `id="tabNav"`**：`renderTabNav` 通过此 ID 查找容器
2. **主内容区需留出底部空间**：设置 `padding-bottom: 70px`，避免内容被固定导航栏遮挡
3. **每个页面都要调用 `renderTabNav`**：因为是独立页面，每个页面需要自行渲染导航栏
4. **导航栏最大宽度 600px**：与 `.app-container` 一致，居中显示
5. **z-index: 100**：确保导航栏在页面内容之上，但在弹窗（z-index: 10002）之下
