# 用户端 - 添加车辆（ppz-m/my-cars/add/）

## 一、页面概述

添加车辆页面提供车辆信息录入表单，支持新增车辆和编辑已有车辆两种模式。提交后进入审核流程。

**访问路径**：
- 新增：`/static/ppz-m/my-cars/add/index.html`
- 编辑：`/static/ppz-m/my-cars/add/index.html?audit_id=xxx`

**前置条件**：用户已登录且司机状态正常

## 三、功能模块

### 3.1 表单字段

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| 车辆型号 | text | 是 | 非空 | 如：奔驰GLC300L |
| 行驶证照片 | text | 是 | 非空 | 行驶证照片URL |
| 驾驶证照片 | text | 是 | 非空 | 驾驶证照片URL |
| 乘客座位数 | number | 是 | 大于0 | 如：5 |
| 车辆生活照 | text | 是 | 非空 | 车辆照片URL |
| 车辆简介 | textarea | 是 | 非空 | 车辆描述文字 |

### 3.2 新增模式

**页面标题**：新增车辆

**流程**：
```
填写表单 → 点击「提交审核」→ 前端校验 → 调用 /ppz/addMyCar
  → 成功 → Toast提示 → 返回车辆列表
  → 失败 → 显示错误信息
```

**后端处理**：
1. 校验司机状态（是否被封禁）
2. 校验必填字段
3. 创建审核记录（audit_status=1, car_id=0）
4. 写入审核日志
5. 自动创建用户扩展记录（如不存在）

### 3.3 编辑模式

**页面标题**：编辑车辆

**触发条件**：URL 中携带 `audit_id` 参数

**流程**：
```
页面加载 → 根据 audit_id 获取车辆详情 → 回填表单
→ 修改表单 → 点击「提交审核」→ 调用 /ppz/editMyCar
  → 成功 → Toast提示 → 返回车辆列表
  → 失败 → 显示错误信息
```

**后端处理**：
1. 校验司机状态
2. 校验必填字段
3. 更新审核记录的 audit_data
4. 重置 audit_status = 1（待审核）
5. 写入审核日志

## 四、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /ppz/addMyCar | 添加车辆 | 新增模式提交 |
| POST /ppz/editMyCar | 编辑车辆 | 编辑模式提交 |
| POST /ppz/getMyCarDetail | 获取车辆详情 | 编辑模式页面加载 |

**请求参数**（addMyCar）：
```json
{
  "car_model": "奔驰GLC300L",
  "license_photo": "https://...",
  "driver_photo": "https://...",
  "seats": 5,
  "car_photo": "https://...",
  "description": "这是一辆奔驰GLC300L..."
}
```

**请求参数**（editMyCar）：
```json
{
  "audit_id": 1,
  "car_model": "奔驰GLC300L",
  "license_photo": "https://...",
  "driver_photo": "https://...",
  "seats": 5,
  "car_photo": "https://...",
  "description": "更新后的描述..."
}
```

## 五、业务规则

1. **司机状态校验**：被封禁的司机无法添加或编辑车辆
2. **审核重置**：编辑车辆后，审核状态重置为待审核，需管理员重新审核
3. **数据不丢失**：编辑时更新的是审核表的 audit_data，原车辆表数据在审核通过前不会变更
4. **URL 参数优先级**：如果同时提供 audit_id 和 car_id，优先使用 audit_id
