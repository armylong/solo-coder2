# 用户端 - 登录组件（common/login-modal）

## 一、组件概述

登录弹窗组件是全局公共组件，提供登录/注册功能。任何页面需要登录时，调用 `requireLogin(callback)` 即可：已登录直接执行回调，未登录弹出登录弹窗，登录成功后自动执行回调。

**设计理念**：登录是全局行为，不应绑定在某个页面内。弹窗方式让用户留在当前页面，不丢失上下文。

## 三、API

### 3.1 导出方法

| 方法 | 参数 | 说明 |
|------|------|------|
| `requireLogin(callback)` | callback: Function | 登录守卫。已登录直接执行 callback，未登录弹登录弹窗，登录成功后自动执行 callback |

### 3.2 使用示例

```javascript
import { requireLogin } from '/static/common/login-modal.js';

// 场景1：点击需要登录的Tab
requireLogin(() => {
    window.location.href = '/static/ppz-m/profile/';
});

// 场景2：点击需要登录的功能按钮
requireLogin(() => {
    openOrderPage();
});

// 场景3：不需要回调，只触发登录
requireLogin();
```

## 四、弹窗功能

### 4.1 登录表单

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 账号 | text | 是 | 用户名 |
| 密码 | password | 是 | 登录密码 |

**流程**：
```
输入账号密码 → 提交 → 调用 /auth/login
  → 成功 → 存储 token → 关闭弹窗 → 执行回调
  → 失败 → 显示错误 Toast
```

### 4.2 注册表单

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 用户名 | text | 是 | 注册用户名 |
| 密码 | password | 是 | 设置密码 |
| 确认密码 | password | 是 | 再次输入密码 |

**流程**：
```
输入用户名+密码+确认密码 → 提交 → 调用 /auth/register
  → 成功 → Toast 提示"注册成功，请登录" → 切换到登录表单（自动填入用户名）
  → 失败 → 显示错误 Toast
```

### 4.3 表单切换

- 登录表单底部有「立即注册」链接，点击切换到注册表单
- 注册表单底部有「去登录」链接，点击切换到登录表单

## 五、交互行为

### 5.1 打开弹窗

- 调用 `requireLogin()` 时，如果未登录，弹窗从底部弹出
- 弹窗打开时，背景页面添加毛玻璃效果

### 5.2 关闭弹窗

| 方式 | 说明 |
|------|------|
| 点击关闭按钮（×） | 关闭弹窗，不执行回调 |
| 点击遮罩层 | 关闭弹窗，不执行回调 |
| 登录成功 | 自动关闭弹窗，执行回调 |

### 5.3 动画效果

| 动画 | 参数 | 说明 |
|------|------|------|
| 遮罩层淡入 | opacity 0→1, 0.3s ease | 半透明遮罩渐显 |
| 卡片弹入 | translateY(20px)→0 + scale(0.95)→1, 0.3s cubic-bezier | 弹性缩放动画 |
| 关闭淡出 | opacity 1→0, 0.3s ease + 300ms 后移除 DOM | 渐隐后清理 |

## 七、集成方式

### 7.1 HTML 引入

在需要使用登录弹窗的页面中引入 CSS：

```html
<link rel="stylesheet" href="/static/common/login-modal.css">
```

弹窗 DOM 由 JS 动态创建，无需在 HTML 中预写结构。

### 7.2 JS 引入

```javascript
import { requireLogin } from '/static/common/login-modal.js';
```

### 7.3 依赖

| 依赖 | 说明 |
|------|------|
| common/auth.js | Token 管理、API 调用 |
| common/session-data.js | 会话数据缓存层（按需获取业务数据） |
| common/ui.js | showLoading / hideLoading / showToast |
| common/style.css | 通用样式（form-group 等） |

## 八、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /auth/login | 用户登录 | 登录表单提交 |
| POST /auth/register | 用户注册 | 注册表单提交 |

---

## 九、注意事项

1. **回调只执行一次**：登录成功后执行回调并清空，不会重复执行
2. **Token 自动管理**：登录成功后自动存储 token 到 Cookie
3. **业务数据按需获取**：各页面通过 SessionData.fetch 按需获取，不再登录时预取
4. **DOM 自动清理**：弹窗关闭后 300ms 自动移除 DOM，避免内存泄漏
5. **多次调用安全**：每次调用 `requireLogin` 都会创建新的弹窗实例
