# 用户端 - 车辆管理（ppz-m/my-cars/）

## 一、页面概述

车辆管理页面展示用户已添加的所有车辆信息，支持查看车辆照片轮播、添加新车辆、查看车辆详情等操作。

**访问路径**：`/static/ppz-m/my-cars/index.html`

**前置条件**：用户已登录

---

## 三、功能模块

### 3.1 车辆照片轮播

- 展示所有已通过审核的车辆生活照
- 支持左右滑动切换
- 底部圆点指示器显示当前位置

### 3.2 车辆列表

**列表项展示**：
| 信息 | 说明 |
|------|------|
| 车辆型号 | 如：奔驰GLC300L |
| 座位数 | 乘客座位数 |
| 审核状态 | 待审核/已通过/已驳回 |

**审核状态标识**：
| 状态 | 显示 | 颜色 |
|------|------|------|
| 待审核 | 待审核 | 黄色 |
| 已通过 | 已通过 | 绿色 |
| 已驳回 | 已驳回 | 红色 |

### 3.3 添加车辆

**入口**：列表标题旁「+ 添加车辆」按钮

**跳转**：`/static/ppz-m/my-cars/add/`

## 四、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /ppz/getMyCars | 获取车辆列表 | 页面加载 |
| POST /ppz/checkDriver | 检查是否认证司机 | 页面加载 |

**请求参数**（getMyCars）：
```json
{
  "audit_status": 0
}
```

**响应数据**：

## 五、业务规则

1. **司机状态校验**：如果司机已被封禁，页面会提示无法管理车辆
2. **审核状态筛选**：默认展示所有状态的车辆（audit_status=0）
3. **照片轮播**：仅展示已通过审核（audit_status=2）的车辆照片
4. **空状态**：无车辆时显示空状态提示，引导用户添加车辆
