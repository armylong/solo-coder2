# 用户端 - 车辆详情（ppz-m/my-cars/detail/）

## 一、页面概述

车辆详情页面展示单辆车辆的完整信息，包括车辆照片、型号、座位数、审核状态等。支持编辑和删除操作。

**访问路径**：`/static/ppz-m/my-cars/detail/index.html?audit_id=xxx` 或 `?car_id=xxx`

**前置条件**：用户已登录

---

## 三、功能模块

### 3.1 车辆信息展示

| 信息项 | 来源字段 | 说明 |
|--------|---------|------|
| 车辆照片 | audit_data.car_photo | 点击可放大查看 |
| 车辆型号 | audit_data.car_model | 如：奔驰GLC300L |
| 乘客座位数 | audit_data.seats | 数字 |
| 审核状态 | audit_status | 待审核/已通过/已驳回 |
| 车辆简介 | audit_data.description | 文本 |
| 行驶证照片 | audit_data.license_photo | 点击可放大查看 |
| 驾驶证照片 | audit_data.driver_photo | 点击可放大查看 |

### 3.2 图片放大查看

- 点击车辆照片或证件照片，弹出全屏图片查看弹窗
- 弹窗中显示大图和标题
- 点击弹窗外部或关闭按钮关闭

### 3.3 编辑车辆

**流程**：
```
点击「编辑车辆」→ 跳转添加车辆页面（编辑模式）
  → /my-cars/add/?audit_id=xxx
```

### 3.4 删除车辆

**流程**：
```
点击「删除」→ 确认弹窗 → 调用删除API → 返回车辆列表
```

**注意**：删除是软删除，审核记录的 audit_status 置为 0

## 四、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /ppz/getMyCarDetail | 获取车辆详情 | 页面加载 |
| POST /ppz/deleteMyCar | 删除车辆 | 点击删除确认后 |

**请求参数**（getMyCarDetail）：
```json
{
  "audit_id": 1
}
```
或
```json
{
  "car_id": 1
}
```

**响应数据**：
```json
{
  "car": {
    "audit_id": 1,
    "car_id": 1,
    "uid": 100,
    "car_model": "奔驰GLC300L",
    "license_photo": "https://...",
    "driver_photo": "https://...",
    "seats": 5,
    "car_photo": "https://...",
    "description": "...",
    "audit_status": 2,
    "review_status": 2,
    "audit_reason": "",
    "created_at": "2026-01-01T00:00:00Z"
  }
}
```

## 五、数据查询逻辑

获取车辆详情时，后端按以下优先级查询：

```
1. 如果提供了 audit_id → 直接查询审核表
2. 如果提供了 car_id → 先查审核表（uid + car_id）
   → 审核表有记录 → 返回审核记录
   → 审核表无记录 → 查询车辆表 → 返回车辆信息（audit_status=2）
3. 都未提供 → 返回错误
```

这种设计确保了：
- 优先展示审核记录（包含审核状态信息）
- 如果审核表无记录但车辆表有，说明是已通过的车辆
- 车辆表数据返回时，audit_status 固定为 2（已通过）
