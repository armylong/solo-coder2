# 用户端 - 首页（ppz-m/index.html）

## 一、页面概述

拼拼坐用户端采用 Tab 式多页面结构，每个 Tab 是独立的 HTML 页面，通过底部导航栏切换。登录功能由公共登录弹窗组件提供，不再绑定在某个页面内。

**当前状态**：拼车功能正在建设中，仅「我的」标签页可用。

---

## 二、页面结构

### 2.1 底部导航栏

| Tab | 图标 | 路径 | 需要登录 | 说明 |
|-----|------|------|---------|------|
| 拼车 | 🚗 | /static/ppz-m/ | 否 | 拼车功能入口（建设中） |
| 我的 | 👤 | /static/ppz-m/profile/ | 是 | 个人中心 |

### 2.2 页面流转

```
拼车页（首页）
├── 点击"我的"Tab
│   ├── 已登录 → 跳转我的页
│   └── 未登录 → 弹出登录弹窗 → 登录成功 → 跳转我的页
│
└── 未来：点击需要登录的功能
    └── 未登录 → 弹出登录弹窗 → 登录成功 → 执行回调

我的页
├── 未登录访问 → 跳转拼车首页
├── 已登录 → 个人中心
│   ├── 用户头像 + 昵称 + 手机号
│   ├── 编辑个人信息 → 个人信息页
│   ├── 车辆管理 → /my-cars/
│   ├── 常用地址 → /address/
│   └── 退出登录 → 跳转拼车首页
```

## 三、登录弹窗组件

登录弹窗是公共组件（`common/login-modal.js`），任何页面都可以调用。

### 3.1 调用方式

```javascript
import { requireLogin } from '/static/common/login-modal.js';

requireLogin(() => {
    // 登录后执行的操作
});
```

- **已登录**：直接执行回调
- **未登录**：弹出登录弹窗，登录成功后自动执行回调

### 3.2 弹窗功能

| 功能 | 说明 |
|------|------|
| 登录 | 输入账号+密码，调用 /auth/login |
| 注册 | 输入用户名+密码+确认密码，调用 /auth/register |
| 切换 | 登录/注册表单切换 |
| 关闭 | 点击遮罩或关闭按钮 |

### 3.3 弹窗样式

- 毛玻璃背景（backdrop-filter: blur）
- 半透明遮罩（rgba(0,0,0,0.3)）
- 圆角卡片 + 渐变按钮
- 弹性缩放动画

**按时出发/按时到达切换**：点击标题行弹出选项，切换后标题和弹性文案跟着变。

**时间滚轮选择器**：点击时间行弹出底部三列滚轮：
- 日期列：今天/明天/后天/周x，显示未来7天
- 小时列：00-23
- 分钟列：00-59
- 中间高亮行 + 上下 padding 实现居中对齐
- scroll-snap 吸附 + scrollend 自动对齐

**弹性时间选项**：点击弹性行弹出选项：
| 选项 | 值 |
|------|---|
| 10分钟 | 10 |
| 30分钟 | 30 |
| 1小时 | 60 |
| 3小时 | 180 |

按时出发时文案"最多等X"，按时到达时文案"可提前X"。

### 4.3 地址选择弹窗

点击出发地或目的地输入框，底部弹出地址选择弹窗。出发地和目的地共用同一个弹窗，根据场景决定是否显示"我的位置"。

**选择出发地时**：
```
┌──────────────────────────┐
│  ─ ─ ─                   │  ← 拖拽条
│                           │
│  🔍 搜索地址              │
├───────────────────────────┤
│                           │  ← 整体滚动
│  📍 我的位置        1.2km │  ← 仅出发地显示
│                           │
│  最近使用                 │
│  📍 阳光小区3号楼  1.2km  │
│  📍 XX大厦A座     3.5km  │
│  📍 XX大学         5.1km  │
│                           │
│  常用地址                 │
│  🏠 我家           1.2km  │
│  🏢 公司           3.5km  │
│  🏫 学校           5.1km  │
│                           │
├───────────────────────────┤  ← 分割线
│  🗺️ 从地图选择            │  ← 固定底部
└───────────────────────────┘
```

**选择目的地时**：不显示"我的位置"，其余相同。

**搜索时**：弹窗内容切换为搜索结果列表，复用高德 POI 搜索。

**交互规则**：
| 操作 | 行为 |
|------|------|
| 点击地址项 | 直接选中，弹窗关闭，卡片回填 |
| 搜索 | 输入关键词，300ms 防抖调高德 POI |
| 搜索结果点选 | 直接选中，弹窗关闭，卡片回填 |
| 从地图选择 | 跳全屏地图页（mode=pick），选完 postMessage 回来 |
| 已选地址右侧 ✕ | 清除选择，恢复占位符 |
| 我的位置 | 调用定位获取当前位置，仅出发地显示 |
| 最近使用 | 从最近 3 条订单取出发地/目的地 |
| 常用地址 | 从 tb_ppz_map_address 取，不区分类型 |
| 距离显示 | 基于当前定位计算直线距离 |

**弹窗样式**：
- 最大高度 70vh，内容超出整体滚动
- "从地图选择"固定在底部，不随滚动
- 搜索框始终固定在顶部

## 五、我的页（Tab 2）

**路径**：`/static/ppz-m/profile/`

**前置条件**：必须已登录，未登录访问自动跳转拼车首页。

### 5.1 个人中心

**展示信息**：
- 用户头像（取用户名首字母）
- 用户昵称
- 手机号（未绑定时显示"未绑定手机号"）
- 认证司机标签（car_count > 0 时显示）

**功能菜单**：
| 菜单 | 图标 | 跳转 |
|------|------|------|
| 车辆管理 | 🚗 | /static/ppz-m/my-cars/ |
| 常用地址 | 📍 | /static/ppz-m/address/ |
| 退出登录 | 🚪 | 清除登录态，跳转拼车首页 |

### 5.2 个人信息编辑

**入口**：个人中心头像旁「编辑」按钮

**可编辑字段**：
| 字段 | 说明 | 编辑方式 |
|------|------|---------|
| 账号 | 只读 | - |
| 昵称 | 可编辑 | 弹窗编辑 |
| 手机号 | 可编辑 | 弹窗编辑 |
| 邮箱 | 可编辑 | 弹窗编辑 |

**修改密码**：
- 需输入旧密码 + 新密码 + 确认新密码
- 修改成功后清除登录态，跳转拼车首页

## 六、API 调用

| API | 说明 | 调用时机 |
|-----|------|---------|
| POST /auth/login | 用户登录 | 登录弹窗提交 |
| POST /auth/register | 用户注册 | 登录弹窗注册提交 |
| POST /auth/logout | 退出登录 | 我的页退出登录 |
| SessionData.fetch(['user', 'ppz_user']) | 获取用户和拼拼坐数据 | 我的页加载时 |
| SessionData.fetch(['ppz_user']) | 获取拼拼坐数据 | 首页/抢单页加载时 |
| POST /user/updateUserInfo | 更新用户信息 | 编辑个人信息后提交 |
| POST /user/changePassword | 修改密码 | 修改密码弹窗提交 |
| POST /ppz getAddressPickerData | 获取地址选择器数据 | 打开地址选择弹窗时 |
| POST /ppz createOrder | 创建拼车订单 | 确认拼车提交 |

---

## 七、交互说明

1. **登录态持久化**：登录成功后将 token 存储在 Cookie 中，页面加载时自动检查登录态
2. **登录弹窗**：未登录时点击需要登录的功能，弹出登录弹窗，登录成功后自动执行原操作
3. **我的页守卫**：未登录直接访问我的页，自动跳转拼车首页
4. **退出登录**：退出后跳转拼车首页，不再停留在我的页
5. **Toast 提示**：所有操作结果通过 Toast 组件展示，自动消失
6. **Loading 状态**：API 请求期间显示全局 Loading 遮罩
