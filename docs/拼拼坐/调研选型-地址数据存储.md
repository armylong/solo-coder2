# 调研选型 - 地址数据存储

## 一、背景

`tb_ppz_map_address` 表的 `gaode_data` 字段存储高德地图返回的完整 JSON 数据，数据量过大，需要精简存储。

## 三、前端使用情况

| 使用场景 | 用到的字段 |
|---------|-----------|
| 地址列表页 | name、address（展示地址名称和详细地址） |
| 地图编辑页 | location（经纬度定位） |
| 未来派车功能 | lng、lat（判断用户是否在运营区域内）、adcode（快速预筛） |

## 四、字段筛选讨论

### 4.1 formatted_address

- 高德拼好的完整地址，列表页直接展示
- 如果不存，需要自己从 province+city+district+township+... 拼接，直辖市、空数组等边界情况容易出错
- **决定：存**

### 4.2 location 格式

三种方案：
- 字符串 `"116.397,39.909"`：每次用都得 split 解析
- 数组 `[116.397, 39.909]`：紧凑，但 lng/lat 谁前谁后容易混淆
- 俩字段 `lng: 116.397, lat: 39.909`：最直观，不用解析，不歧义

**决定：拆成 lng 和 lat 两个独立字段**

### 4.3 township

- 信息已包含在 formatted_address 中
- 但未来派车功能可能需要按街道级别匹配
- **决定：存**

### 4.4 aoi_name

- AOI = Area of Interest，兴趣面（如小区、商场的区域边界）
- `regeocode.aois[0].name` 表示用户所在的最小区域名称（如"裕龙花园六区"）
- 派车时可能需要更精确的位置描述
- 部分位置可能命中不到 AOI，此时存空字符串
- **决定：存，取 aois 第0个的 name**

### 4.5 province/city/district/adcode

- 列表页暂时只用 formatted_address 展示
- 但派车功能需要 adcode 做快速预筛，province/city/district 可能用于区域匹配
- **决定：存**

### 4.6 不存的字段

| 字段 | 原因 |
|------|------|
| status/info/infocode | 高德接口状态信息，无业务价值 |
| streetNumber | 已包含在 formatted_address 中 |
| businessAreas | 商圈信息，暂无业务场景 |
| building | 建筑信息，暂无业务场景 |
| neighborhood | 已包含在 formatted_address 中 |
| citycode | 暂无业务场景 |
| towncode | 已有 adcode，暂无业务场景 |

## 五、最终方案

精简后的 `gaode_data` 存储 9 个字段：

| 字段 | 类型 | 示例 | 来源 |
|------|------|------|------|
| `formatted_address` | string | "北京市顺义区光明街道裕龙花园(六区)滨河小区" | regeocode.formatted_address |
| `lng` | float | 116.677018 | 逆地理编码时的中心点经度 |
| `lat` | float | 40.118801 | 逆地理编码时的中心点纬度 |
| `province` | string | "北京市" | addressComponent.province |
| `city` | string | "北京市" | addressComponent.city（空数组时取 province） |
| `district` | string | "顺义区" | addressComponent.district |
| `township` | string | "光明街道" | addressComponent.township |
| `adcode` | string | "110113" | addressComponent.adcode |
| `aoi_name` | string | "裕龙花园六区" | regeocode.aois[0].name（无 AOI 时为空字符串） |

实际存储示例：

```json
{
  "formatted_address": "山东省泰安市新泰市泉沟镇兴泉家园",
  "lng": 117.643879,
  "lat": 35.981911,
  "province": "山东省",
  "city": "泰安市",
  "district": "新泰市",
  "township": "泉沟镇",
  "adcode": "370982",
  "aoi_name": "兴泉家园"
}
```

## 六、兼容性处理

- `parseGaodeData` 函数新增对精简格式的识别：有 `formatted_address` 字段走新分支
- 旧数据（regeocode 完整格式）走原有解析逻辑，不受影响
- 精简逻辑 `compactGaodeData` 放在 address-map 的 main.js 中，不修改公共模块 gaode-map.js
