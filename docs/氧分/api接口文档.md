# 氧分模块 API 接口文档

## 1. 概述

### 1.1 基础信息

- **协议**: HTTP/HTTPS
- **Content-Type**: application/json
- **认证方式**: 需要登录认证（通过Cookie或Token）
- **路由前缀**: `/yangfen`
- **字符编码**: UTF-8

### 1.2 通用响应结构

#### 成功响应

```json
{
    "code": 0,
    "message": "",
    "data": {
        "uid": "123",
        "balance": 100
    }
}
```

#### 错误响应

```json
{
    "code": -401,
    "message": "请先登录",
    "data": null
}
```

### 1.3 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，>=0 表示成功（0 为通用成功），<0 表示失败（-1 为通用失败） |
| message | string | 状态信息，成功时为空字符串 |
| data | object/any | 响应数据，具体结构取决于接口 |

### 1.4 接口调用方式

**路由注册**: 控制器方法以 `Action` 开头，路由会自动转为小写，去掉 `Action` 前缀。

例如:
- `ActionGetBalance` → 路由: `POST /yangfen/getBalance`
- `ActionRecharge` → 路由: `POST /yangfen/recharge`

**前端调用示例** (`static/yangfen/index.html`):
```javascript
const API_BASE = 'http://localhost/yangfen';

// 调用方式
async function apiCall(action, params = {}) {
    const data = await Auth.fetchApi(API_BASE, action, params);
    return data;
}

// 示例: 查询余额
const balanceData = await apiCall('getBalance', {});

// 示例: 充值
await apiCall('recharge', { amount: 100, expire_sec: 86400 });
```

## 2. 接口详情

### 2.1 查询余额

**接口地址**: `POST /yangfen/getBalance`

**接口描述**: 查询当前登录用户的氧分余额

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 无 | - | - | - |

**请求示例**:
```json
{}
```

**响应示例**:
```json
{
    "code": 0,
    "message": "",
    "data": {
        "uid": "123",
        "balance": 100
    }
}
```

**响应字段说明**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| uid | string | 用户ID |
| balance | int | 当前余额 |

---

### 2.2 充值

**接口地址**: `POST /yangfen/recharge`

**接口描述**: 为用户账户充值氧分

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| amount | int | 是 | 充值金额，必须大于0 |
| expire_sec | int64 | 否 | 过期时间（秒），0表示永不过期，默认值参考前端实现 |

**请求示例**:
```json
{
    "amount": 100,
    "expire_sec": 86400
}
```

**响应示例**:
```json
{
    "code": 0,
    "message": "",
    "data": {
        "uid": "123",
        "balance": 200
    }
}
```

**前端快速充值按钮**:
- `+50` - 调用 `recharge` 接口，amount=50
- `+100` - 调用 `recharge` 接口，amount=100
- `+500` - 调用 `recharge` 接口，amount=500

---

**接口地址**: `POST /yangfen/consume`

**接口描述**: 从用户账户消费氧分

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| amount | int | 是 | 消费金额，必须大于0 |

**请求示例**:
```json
{
    "amount": 50
}
```

**响应示例**:
```json
{
    "code": 0,
    "message": "",
    "data": {
        "uid": "123",
        "balance": 150
    }

```

**前端快速消费按钮**:
- `-10` - 调用 `consume` 接口，amount=10
- `-50` - 调用 `consume` 接口，amount=50
- `-100` - 调用 `consume` 接口，amount=100

---

### 2.4 转账

**接口地址**: `POST /yangfen/transfer`

**接口描述**: 从当前用户账户向其他用户转账氧分

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| toUid | string | 是 | 目标用户ID |
| amount | int | 是 | 转账金额，必须大于0 |

**请求示例**:
```json
{
    "toUid": "456",
    "amount": 30
}
```

**响应示例**:
```json
{
    "code": 0,
    "message": "",
    "data": {
        "success": true,
        "message": "转账成功",
        "data": {
            "fromUid": "123",
            "fromBalance": 120,
            "toUid": "456",
            "toBalance": 80
        }
    }
}
```

---

### 2.5 退款

**接口地址**: `POST /yangfen/refund`

**接口描述**: 对指定的消费交易进行退款

**注意**: 前端页面中，交易记录列表里的"消费"类型记录旁边会显示"退款"按钮，点击可直接调用此接口。

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| transactionId | string | 是 | 交易ID |

**请求示例**:
```json
{
    "transactionId": "TX1714000000000000000"
}
```

**响应示例**:
```json
{
    "code": 0,
    "message": "",
    "data": {
        "uid": "123",
        "balance": 170
    }
}
```

---

### 2.6 查询交易记录

**接口地址**: `POST /yangfen/getTransactions`

**接口描述**: 查询当前登录用户的交易记录（最近100条）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 无 | - | - | - |

**请求示例**:
```json
{}
```

**响应示例**:
```json
{
    "code": 0,
    "message": "",
    "data": {
        "list": [
            {
                "id": "TX1714000000000000000",
                "uid": "123",
                "type": "recharge",
                "amount": 100,
                "balance": 100,
                "description": "充值",
                "createdAt": 1714000000
            },
            {
                "id": "TX1714000001000000000",
                "uid": "123",
                "type": "consume",
                "amount": 50,
                "balance": 50,
                "description": "消费50积分",
                "createdAt": 1714000001
            }
        ],
        "total": 2
    }
}
```

**响应字段说明**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| list | array | 交易记录列表 |
| list[].id | string | 交易ID |
| list[].uid | string | 用户ID |
| list[].type | string | 交易类型 |
| list[].amount | int | 交易金额 |
| list[].balance | int | 交易后余额 |
| list[].description | string | 交易描述 |
| list[].createdAt | int64 | 创建时间（Unix时间戳） |
| total | int | 记录总数 |

**前端显示**:
- 交易类型会显示为中文：充值、消费、转出、转入、退款
- 金额显示会根据类型添加 `+` 或 `-` 前缀
- 消费类型的记录会显示"退款"按钮

## 3. 交易类型说明

| 类型代码 | 中文显示 | 余额变化方向 | 前端显示颜色 |
|----------|----------|--------------|--------------|
| recharge | 充值 | +（增加） | 绿色 |
| consume | 消费 | -（减少） | 红色 |

## 4. 前端页面说明

### 4.1 页面功能

**位置**: `static/yangfen/index.html`

这是一个氧分系统的测试页面，提供了完整的可视化操作界面：

1. **顶部信息栏**:
   - 显示当前用户ID
   - 显示当前氧分余额

2. **充值卡片**:
   - 输入充值金额
   - 输入过期时间（秒），默认86400秒（24小时）
   - 快速充值按钮：+50, +100, +500

3. **消费卡片**:
   - 输入消费金额
   - 快速消费按钮：-10, -50, -100

4. **转账卡片**:
   - 输入目标用户ID
   - 输入转账金额

5. **退款卡片**:
   - 输入交易号
   - 提示：可以在交易记录中点击退款按钮快速操作

6. **交易记录列表**:
   - 显示所有交易记录
   - 显示交易类型（中文）
   - 显示金额（带正负号，颜色区分）
   - 显示时间
   - 消费类型记录显示"退款"按钮

7. **结果面板**:
   - 操作结果会以JSON格式显示在底部面板
   - 成功时显示绿色背景
   - 失败时显示红色背景

### 4.2 页面依赖

- `/static/common/auth.js` - 提供认证和API调用功能
- 需要用户先登录才能使用

## 5. 错误码说明

| 状态码 | 说明 |
|--------|------|
| 0 | 成功 |
| -401 | 未登录或Token无效 |
| -1 | 通用失败 |
| 其他 | 业务错误，具体见message |

**常见业务错误信息**:
- "请先登录" - 未登录访问
- "充值金额必须大于0" - amount <= 0
- "消费金额必须大于0" - amount <= 0
- "用户不存在" - 未找到用户余额记录
- "余额不足" - 余额小于操作金额
- "目标用户不能为空" - toUid为空
- "不能转给自己" - fromUid == toUid
- "交易号不能为空" - transactionId为空
- "交易记录不存在" - 未找到交易记录
- "只能退款消费记录" - 交易类型不是consume

## 6. 接口问题与风险

### 6.1 已知问题

1. **并发安全问题**: 所有涉及余额修改的接口（充值、消费、转账、退款）都没有并发控制，高并发下可能出现数据不一致。

2. **无事务保护**: 转账操作涉及两个用户的余额修改和两条交易记录，没有事务保护，可能出现部分成功的情况。

3. **退款安全漏洞**: 
   - 没有验证交易记录是否属于当前用户
   - 积分过期后仍可退款，导致用户凭空获得积分
   - 同一交易ID可多次退款

4. **无幂等性**: 所有写操作接口没有幂等性保证，重复请求可能导致重复操作。

5. **交易记录不应被删除**: 交易记录是重要的审计日志，当前代码中 `ClearData` 方法会删除所有交易记录，这在生产环境中是不合适的。
