**方案A: 乐观锁（推荐）**
```go

// 更新时检查版本号

    sql := `UPDATE tb_yangfen_balance SET balance = ?, version = version + 1, updated_at = CURRENT_TIMESTAMP WHERE uid = ? AND version = ?`
    result, err := sqlite.DB.DB().Exec(sql, balance, uid, version)
    if err != nil {
        return false, err
    }
    rowsAffected, _ := result.RowsAffected()
    return rowsAffected > 0, nil
}
```

**方案B: 应用层锁**
```go
var (
    userLocks sync.Map // key: uid, value: *sync.Mutex
)

func getUserLock(uid string) *sync.Mutex {
    lock, _ := userLocks.LoadOrStore(uid, &sync.Mutex{})
    return lock.(*sync.Mutex)
}

func (b *yangfenBusiness) Transfer(ctx context.Context, fromUid, toUid string, amount int) error {
    // 按顺序获取锁，避免死锁
    lock1, lock2 := getUserLock(fromUid), getUserLock(toUid)
    if fromUid < toUid {
        lock1.Lock()
        lock2.Lock()
    } else {
        lock2.Lock()
        lock1.Lock()
    }
    defer lock1.Unlock()
    defer lock2.Unlock()
    
    // 业务逻辑...
}
```

**预期收益**:
- 消除竞态条件
- 保证数据一致性
- 支持高并发场景

## 3. 功能增强

### 3.1 完善过期机制

**当前状态**: 过期机制简单，只清零余额，没有记录过期操作。

**⚠️ 重要原则**:
- **交易记录是审计日志，不应该被删除**
- 过期操作应该记录一条新的交易记录，而不是删除历史记录

**优化建议**:

```go
type ExpireStrategy int

const (
    ExpireStrategyClearBalance ExpireStrategy = iota  // 只清零余额（推荐）
    // ExpireStrategyClearAll 已废弃 - 不应该删除交易记录
)

type ExpireConfig struct {
    Strategy        ExpireStrategy
    RecordExpireTx  bool  // 是否记录过期交易（推荐: true）
    NotifyUser      bool  // 是否通知用户
}

func (b *yangfenBusiness) checkAndClearExpired(ctx context.Context, uid string, config ExpireConfig) error {
    row, err := yangfenModel.TbYangfenBalanceModel.GetByUid(uid)
    if err != nil {
        return nil
    }
    
    if row.ExpireTime > 0 && time.Now().Unix() > row.ExpireTime {
        oldBalance := row.Balance
        
        if oldBalance > 0 {
            // 清零余额
            yangfenModel.TbYangfenBalanceModel.UpdateBalance(uid, 0)
            
            // ✅ 正确做法: 记录"过期清零"的交易记录
            // 交易记录应该保留，不能删除
            if config.RecordExpireTx {
                b.addTransaction(ctx, uid, "expire", oldBalance, 0, 
                    fmt.Sprintf("积分过期清零: %d", oldBalance))
            }
        }
    }
    return nil
}
```

**关于交易记录的设计原则**:
| 原则 | 说明 |
|------|------|
| ✅ 永久保留 | 交易记录是审计日志，应该永久保留 |
| ✅ 记录所有变动 | 每个余额变动都应该有对应的交易记录 |
| ✅ 过期也是变动 | 过期清零也应该记录一条交易记录 |
| ❌ 不删除 | 不应该物理删除任何交易记录 |
| ❌ 不清零 | 不应该"清理"历史交易记录 |

**预期收益**:
- 过期策略可配置
- 数据处理更合理
- 审计日志完整

### 4.2 添加限流和防刷机制

**当前状态**: 没有限流，可能被恶意请求攻击。

**优化建议**:

```go
import (
    "golang.org/x/time/rate"
    "sync"
)

type RateLimiter struct {
    limiters map[string]*rate.Limiter
    mu       sync.Mutex
    r        rate.Limit
    b        int
}

func NewRateLimiter(r rate.Limit, b int) *RateLimiter {
    return &RateLimiter{
        limiters: make(map[string]*rate.Limiter),
        r:        r,
        b:        b,
    }
}

func (rl *RateLimiter) GetLimiter(uid string) *rate.Limiter {
    rl.mu.Lock()
    defer rl.mu.Unlock()
    
    limiter, exists := rl.limiters[uid]
    if !exists {
        limiter = rate.NewLimiter(rl.r, rl.b)
        rl.limiters[uid] = limiter
    }
    return limiter
}

// 使用示例
var transferLimiter = NewRateLimiter(rate.Limit(1), 5) // 每秒1次，突发5次

func (b *yangfenBusiness) Transfer(ctx context.Context, fromUid, toUid string, amount int) error {
    // 限流检查
    limiter := transferLimiter.GetLimiter(fromUid)
    if !limiter.Allow() {
        return fmt.Errorf("操作过于频繁，请稍后再试")
    }
    // ...
}
```

**预期收益**:
- 防止恶意刷分

### 7.2 添加缓存层

**当前状态**: 每次查询都直接访问数据库。

**优化建议**:

```go
import (
    "sync"
    "time"
)

type BalanceCache struct {
    data      map[string]cacheItem
    mu        sync.RWMutex
    ttl       time.Duration
}

type cacheItem struct {
    balance   int
    timestamp time.Time
}

func NewBalanceCache(ttl time.Duration) *BalanceCache {
    return &BalanceCache{
        data: make(map[string]cacheItem),
        ttl:  ttl,
    }
}

func (c *BalanceCache) Get(uid string) (int, bool) {
    c.mu.RLock()
    defer c.mu.RUnlock()
    
    item, exists := c.data[uid]
    if !exists {
        return 0, false
    }
    
    if time.Since(item.timestamp) > c.ttl {
        delete(c.data, uid)
        return 0, false
    }
    
    return item.balance, true
}

func (c *BalanceCache) Set(uid string, balance int) {
    c.mu.Lock()
    defer c.mu.Unlock()
    
    c.data[uid] = cacheItem{
        balance:   balance,
        timestamp: time.Now(),
    }
}

func (c *BalanceCache) Delete(uid string) {
    c.mu.Lock()
    defer c.mu.Unlock()
    delete(c.data, uid)
}
```

**预期收益**:
- 减少数据库压力
- 提升查询性能
- 更好的可扩展性

## 8. 关于交易记录的重要说明

### 8.1 设计原则

在优化过程中，必须遵循以下原则：

| 原则 | 说明 | 理由 |
|------|------|------|
| **永久保留** | 交易记录不应该被物理删除 | 审计要求、合规要求 |
| **完整记录** | 每个余额变动都应该有记录 | 数据一致性、可追溯性 |
| **只读为主** | 交易记录应该是只读的 | 防止数据篡改 |
| **顺序存储** | 交易记录应该按时间顺序存储 | 便于查询和分析 |
