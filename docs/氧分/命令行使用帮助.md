# 氧分模块命令行使用帮助

## 1. 命令概述

### 1.1 命令名称

```
armylong-go yangfen
```

或

```
./armylong-go yangfen
```

### 1.2 功能说明

通过命令行方式管理用户氧分，支持余额查询、充值、消费、转账、退款、查询交易记录等操作。

---

## 2. 全局参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| --uid | string | 用户ID |
| --amount | int | 金额（充值/消费/转账时使用） |
| --to-uid | string | 目标用户ID（转账时使用） |
| --expire-sec | int64 | 过期时间（秒，充值时使用） |
| --transaction-id | string | 交易ID（退款时使用） |

---

## 3. 子命令详解

### 3.1 balance - 查询余额

**功能**: 查询指定用户的氧分余额

**用法**:

```bash
armylong-go yangfen balance --uid <用户ID>
```

**参数**:

| 参数 | 必填 | 说明 |
|------|------|------|
| --uid | 是 | 用户ID |

**示例**:

```bash
# 查询用户1的余额
armylong-go yangfen balance --uid 1
```

**输出示例**:

```
用户 1 当前余额: 100
```

---

### 3.2 recharge - 充值

**功能**: 为指定用户充值氧分

**用法**:

```bash
armylong-go yangfen recharge --uid <用户ID> --amount <金额> [--expire-sec <过期秒数>]
```

**参数**:

| 参数 | 必填 | 说明 |
|------|------|------|
| --uid | 是 | 用户ID |
| --amount | 是 | 充值金额，必须大于0 |
| --expire-sec | 否 | 过期时间（秒），0表示永不过期 |

**示例**:

```bash
# 给用户1充值100氧分，永不过期
armylong-go yangfen recharge --uid 1 --amount 100

# 给用户1充值100氧分，24小时后过期
armylong-go yangfen recharge --uid 1 --amount 100 --expire-sec 86400
```

**输出示例**:

```
✓ 充值成功，当前余额: 200
```

**错误输出**:

```
错误: amount 必须大于0
```

---

### 3.3 consume - 消费

**功能**: 从指定用户账户消费氧分

**用法**:

```bash
armylong-go yangfen consume --uid <用户ID> --amount <金额>
```

**参数**:

| 参数 | 必填 | 说明 |
|------|------|------|
| --uid | 是 | 用户ID |
| --amount | 是 | 消费金额，必须大于0 |

**示例**:

```bash
# 从用户1消费50氧分
armylong-go yangfen consume --uid 1 --amount 50
```

**输出示例**:

```
✓ 消费成功，当前余额: 150
```

**错误输出**:

```
消费失败: 余额不足
```

---

### 3.4 transfer - 转账

**功能**: 从一个用户向另一个用户转账氧分

**用法**:

```bash
armylong-go yangfen transfer --uid <转出用户ID> --to-uid <转入用户ID> --amount <金额>
```

**参数**:

| 参数 | 必填 | 说明 |
|------|------|------|
| --uid | 是 | 转出用户ID |
| --to-uid | 是 | 转入用户ID |
| --amount | 是 | 转账金额，必须大于0 |

**示例**:

```bash
# 从用户1向用户2转账30氧分
armylong-go yangfen transfer --uid 1 --to-uid 2 --amount 30
```

**输出示例**:

```
✓ 转账成功
  转出账户 1 余额: 120
  转入账户 2 余额: 80
```

**错误输出**:

```
转账失败: 不能转给自己
```

---

### 3.5 refund - 退款

**功能**: 对指定的消费交易进行退款

**用法**:

```bash
armylong-go yangfen refund --uid <用户ID> --transaction-id <交易ID>
```

**参数**:

| 参数 | 必填 | 说明 |
|------|------|------|
| --uid | 是 | 用户ID |
| --transaction-id | 是 | 交易ID |

**示例**:

```bash
# 对用户1的指定交易进行退款
armylong-go yangfen refund --uid 1 --transaction-id TX1714000000000000000
```

**输出示例**:

```
✓ 退款成功，当前余额: 170
```

**错误输出**:

```
退款失败: 只能退款消费记录
```

### 3.7 clear - 清除数据

**功能**: 清除指定用户的所有氧分数据（余额记录和交易记录）

**⚠️ 注意**:
- 此命令主要用于测试清理，生产环境慎用
- 交易记录是重要的审计日志，不应该被随便删除
- 生产环境中建议只清零余额，保留交易记录

**用法**:

```bash
armylong-go yangfen clear --uid <用户ID>
```

**参数**:

| 参数 | 必填 | 说明 |
|------|------|------|
| --uid | 是 | 用户ID |

**示例**:

```bash
# 清除用户1的所有氧分数据（仅测试环境）
armylong-go yangfen clear --uid 1
```

**输出示例**:

```
✓ 用户 1 数据已清除
```

## 4. 使用示例

### 4.1 完整操作流程示例

```bash
# 1. 给用户1充值200氧分，永不过期
armylong-go yangfen recharge --uid 1 --amount 200

# 2. 查询用户1的余额
armylong-go yangfen balance --uid 1
# 输出: 用户 1 当前余额: 200

# 3. 用户1消费50氧分
armylong-go yangfen consume --uid 1 --amount 50

# 4. 用户1向用户2转账80氧分
armylong-go yangfen transfer --uid 1 --to-uid 2 --amount 80

# 5. 查询用户1的余额
armylong-go yangfen balance --uid 1
# 输出: 用户 1 当前余额: 70

# 6. 查询用户1的交易记录
armylong-go yangfen transactions --uid 1

# 7. 查看用户2的余额
armylong-go yangfen balance --uid 2
# 输出: 用户 2 当前余额: 80
```

### 4.2 过期时间示例

```bash
# 充值100氧分，10秒后过期
armylong-go yangfen recharge --uid 1 --amount 100 --expire-sec 10

# 立即查询余额
armylong-go yangfen balance --uid 1
# 输出: 用户 1 当前余额: 100

# 等待15秒后，查询消费会触发过期检查
sleep 15
armylong-go yangfen consume --uid 1 --amount 10
# 输出: 消费失败: 余额不足（因为已过期清零）

# 查询余额
armylong-go yangfen balance --uid 1
# 输出: 用户 1 当前余额: 0
```

## 5. 注意事项

### 5.1 重要限制

1. **无并发控制**: 命令行工具同样存在并发安全问题，多个命令同时操作同一用户可能导致数据不一致。

2. **无事务保护**: 转账操作没有事务保护，可能出现部分成功的情况。

3. **无幂等性**: 重复执行命令会导致重复操作。

### 5.2 过期机制说明

1. 氧分过期后，余额会被清零，但交易记录不会被删除。

2. 过期检查在以下操作时触发：
   - 查询余额前
   - 充值前
   - 消费前
   - 转账前

3. 转账时，接收方的过期时间会被设为0（永不过期）。

### 5.3 交易记录的重要性

**交易记录是审计日志，应该遵循以下原则**:

✅ **应该做的**:
- 每个余额变动都应该有对应的交易记录
- 过期清零也应该记录一条交易记录
- 交易记录应该永久保留

❌ **不应该做的**:
- 不应该删除任何交易记录
- 不应该"清理"交易记录（测试环境除外）

### 5.4 退出码

当前命令行实现存在问题：所有错误场景都返回退出码0（成功），导致shell脚本无法判断命令是否执行成功。

```bash
# 当前行为（错误）:
armylong-go yangfen consume --uid 1 --amount 999999
# 输出: 消费失败: 余额不足
echo $?  # 返回 0（但实际是失败）

# 期望行为:
echo $?  # 应该返回非0
```

|----------|----------|----------|
| 1 | 所有错误场景都返回退出码0，shell无法判断命令是否成功 | 高 |
