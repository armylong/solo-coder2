# 用户认证系统概览

## 定位

用户认证系统负责：身份验证（你是谁）和权限控制（你能做什么）。

## 架构

### 后端

```
┌──────────────────────────────────────────────────────────┐
│                    HTTP 请求                              │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│  Gin 路由 + 中间件                                       │
│                                                          │
│  Middleware          验证 Token，加载用户到上下文          │
│  RequireAdmin        检查管理员权限                       │
│  RequireSuperAdmin   检查超级管理员权限                   │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│  Controllers (controllers/user/)                         │
│                                                          │
│  AuthController          无需登录                        │
│  UserController          需要登录                        │
│  UserManagementController 需要管理员                     │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│  Business (business/user/)                               │
│                                                          │
│  Login / Logout / Register / ChangePassword / Kickoff    │
│  UserBusiness.Stats / UserList / UpdateStatus / ...      │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│  Model (model/user/)                                     │
│                                                          │
│  TbUserModel / TbUserTokenModel / TbAdminUserModel       │
└──────────────────────────────────────────────────────────┘
```

### 前端

```
┌──────────────────────────────────────────────────────────┐
│  所有页面 (index / api-catcher / long-store / ...)       │
│                                                          │
│  统一调用 Auth.fetchApi(baseUrl, action, params)         │
│  统一调用 Auth.isSuperAdmin() / Auth.isAdmin()           │
│  不再各自实现 fetch / Token / 权限判断                    │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│  static/common/auth.js — 唯一认证入口                    │
│                                                          │
│  PERMISSION 常量          0/1/2 与后端一致               │
│  RESPONSE_CODE 常量       -401/-403 与后端一致           │
│  Auth.fetchApi()          自动带 Token + 处理错误码      │
│  Auth.setToken()          Token 管理                     │
│  Auth.clearToken()                                      │
│  Auth.isSuperAdmin()      权限判断                       │
│  Auth.isAdmin()                                         │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│  后端 API                                                 │
│                                                          │
│  返回 code=0 成功 / code=-401 未登录 / code=-403 无权限 │
└──────────────────────────────────────────────────────────┘
```

## 认证流程

```
客户端                    中间件                     缓存/数据库
  │                        │                           │
  │  请求 + Token          │                           │
  │──────────────────────→ │                           │
  │                        │  查缓存                    │
  │                        │──────────────────────────→│
  │                        │  命中？                    │
  │                        │←──────────────────────────│
  │                        │                           │
  │                        │  [命中] 直接取用户          │
  │                        │  [未命中] 解析 JWT          │
  │                        │  查数据库加载用户           │
  │                        │  写入缓存                   │
  │                        │                           │
  │                        │  设置上下文                 │
  │                        │  ctx.Set("login_user", u)  │
  │                        │  ctx.Set("login_uid", uid) │
  │                        │  ctx.Next()                │
  │                        │                           │
  │  ← 200 + 业务响应      │                           │
  │                        │                           │
```

## 权限等级

| 等级 | 后端常量 | 前端常量 | 值 | 说明 | 可访问 |
|---|---|---|---|---|---|
| 普通用户 | `NORMAL` | `PERMISSION.NORMAL` | 0 | 默认角色 | `/auth/*`, `/user/*` |
| 管理员 | `ADMIN` | `PERMISSION.ADMIN` | 1 | 管理用户 | 以上 + `/user_management/*` |
| 超级管理员 | `SUPER_ADMIN` | `PERMISSION.SUPER_ADMIN` | 2 | 最高权限 | 以上 + 超级管理员专属接口 |

## Token 生命周期

```
登录 → 生成 JWT + 生成随机 Token → 存数据库 + 存缓存 → 返回给客户端
                                                              │
客户端每次请求携带 Token ←────────────────────────────────────┘
         │
         ▼
Auth.fetchApi() 自动带 Token → 后端中间件验证
         │
         ├─ 缓存命中？→ 直接用
         │    未命中 → 解析 JWT → 查数据库 → 回写缓存
         │
         ├─ 返回 -401 → Auth 自动清 Token + 跳首页
         ├─ 返回 -403 → Auth 自动弹窗 + 返回上页
         │
登出/踢下线 → 删除数据库记录 + 删除缓存
```
