# 龙聊 - 业务层设计

## 目录结构

```
business/long_chat/
  chat.go      → ChatBusiness    聊天实体管理
  message.go   → MessageBusiness 消息收发
  friend.go    → FriendBusiness  好友关系
  member.go    → MemberBusiness  会话成员管理
```

## 调用关系

```
HTTP请求 → controllers/long_chat → business/long_chat → model/long_chat
WebSocket消息 → websocket/handler → business/long_chat → model/long_chat
```

两条通道，同一个 business 层。

## ChatBusiness — 聊天实体管理

| 方法 | 说明 |
|---|---|
| CreatePrivateChat(uid, targetUid) | 创建单聊，同时创建双方 member 和 list 记录 |
| CreateGroup(ownerUid, groupName, memberUids) | 创建群聊，同时创建所有 member 和 list 记录 |
| GetChat(chatId) | 获取聊天信息 |
| DismissChat(chatId, operatorUid) | 解散聊天，校验是否为群主 |
| TransferOwner(chatId, operatorUid, newOwnerUid) | 转让群主，校验当前是否为群主 |
| GetOrCreatePrivateChat(uid, targetUid) | 获取或创建单聊（发消息时调用，避免重复创建） |

## MessageBusiness — 消息收发

| 方法 | 说明 |
|---|---|
| SendMessage(chatId, fromUid, msgType, content) | 发送消息 |
| ListMessages(chatId, uid, limit, offset) | 分页获取聊天记录 |
| ClearUnread(chatId, uid) | 清零未读，打开聊天详情时调用 |

### SendMessage 内部流程

1. 校验 chat 状态（是否已解散）
2. 校验发送者是否为成员
3. 插入消息
4. 更新所有参与者的 list（last_msg、last_msg_at）
5. 除发送者外 unread + 1
6. 通过 WebSocket 推送给在线成员

## FriendBusiness — 好友关系

| 方法 | 说明 |
|---|---|
| AddFriend(uid, friendUid) | 添加好友，创建两条记录（发起方 accepted + 被添加方 pending） |
| AcceptFriend(uid, friendUid) | 通过好友请求 |
| RejectFriend(uid, friendUid) | 拒绝好友请求 |
| DeleteFriend(uid, friendUid) | 删除好友，双向删除 |
| ListFriends(uid) | 好友列表 |
| ListPendingRequests(uid) | 待确认请求列表 |
| IsFriend(uid, friendUid) | 是否为好友 |

### AddFriend 双向记录设计

A 加 B 时创建两条记录：
- (A的uid, B的uid, accepted) — A 侧直接通过
- (B的uid, A的uid, pending) — B 侧待确认

B 同意后：(B的uid, A的uid, accepted)

这样 A 和 B 各自查自己的好友列表时，只需 `WHERE uid = ? AND status = accepted`，不用关心方向。

## MemberBusiness — 会话成员管理

| 方法 | 说明 |
|---|---|
| JoinChat(chatId, uid) | 加入聊天，加 member + list 记录 + 系统消息 |
| LeaveChat(chatId, uid) | 退出聊天，删 member + list 记录 + 系统消息 |
| KickMember(chatId, operatorUid, targetUid) | 踢人，校验群主权限 |
| ListMembers(chatId) | 成员列表 |
