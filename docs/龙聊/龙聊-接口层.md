# 龙聊 - 接口层

## 接口总览

所有接口需要登录，请求方式为 POST。

框架自动将 `ActionXxx` 方法映射为路由，规则：首字母小写、驼峰保留。例如 `ActionSendMessage` → `sendMessage`，`ActionGetChatList` → `getChatList`。

接口按 Controller 分组，baseUrl 分别为：
- `/long_chat/chat` — 聊天管理
- `/long_chat/message` — 消息管理
- `/long_chat/friend` — 好友管理
- `/long_chat/member` — 成员管理

---

## 一、聊天管理 `/long_chat/chat`

### 聊天列表 `getChatList`

获取当前用户的聊天列表，按最新消息时间倒序。

**请求**：无参数

**响应**：
```json
{
  "list": [
    {
      "chat_id": "1234567890",
      "chat_type": 1,
      "chat_name": "张三",
      "unread": 3,
      "last_msg": "你好",
      "last_msg_at": "2026-06-10 12:00:00"
    }
  ]
}
```

| 字段 | 说明 |
|---|---|
| chat_type | 1-单聊 2-群聊 |
| chat_name | 单聊为对方昵称，群聊为群名 |
| unread | 未读消息数 |
| last_msg | 最新消息摘要 |
| last_msg_at | 最新消息时间 |

### 创建单聊 `createPrivateChat`

与对方创建或获取已有的单聊，重复调用返回同一 chat_id。

**请求**：
```json
{
  "target_uid": 2
}
```

**响应**：
```json
{
  "chat_id": "1234567890"
}
```

### 创建群聊 `createGroup`

**请求**：
```json
{
  "group_name": "工作群",
  "member_uids": [2, 3, 4]
}
```

**响应**：
```json
{
  "chat_id": "1234567890"
}
```

### 聊天详情 `getChatDetail`

**请求**：
```json
{
  "chat_id": "1234567890"
}
```

**响应**：
```json
{
  "chat_id": "1234567890",
  "chat_type": 2,
  "chat_name": "工作群",
  "owner_uid": 1,
  "status": 1,
  "created_at": "2026-06-10T12:00:00Z",
  "updated_at": "2026-06-10T12:00:00Z"
}
```

### 解散群聊 `dismissChat`

仅群主可操作，单聊不可解散。

**请求**：
```json
{
  "chat_id": "1234567890"
}
```

**响应**：`{}`

### 转让群主 `transferOwner`

仅群主可操作，新群主必须是群成员。

**请求**：
```json
{
  "chat_id": "1234567890",
  "new_owner_uid": 2
}
```

**响应**：`{}`

## 二、消息管理 `/long_chat/message`

### 发送消息 `sendMessage`

发送消息后，后端自动通过 WebSocket 推送给其他在线成员。

**请求**：
```json
{
  "chat_id": "1234567890",
  "msg_type": "text",
  "content": "你好"
}
```

| msg_type | 说明 |
|---|---|
| text | 文本，content 为纯文本 |
| image | 图片，content 为 JSON |
| voice | 语音，content 为 JSON |

**响应**：
```json
{
  "msg": {
    "msg_id": "9876543210",
    "chat_id": "1234567890",
    "from_uid": 1,
    "msg_type": "text",
    "content": "你好",
    "created_at": "2026-06-10T12:00:00Z"
  }
}
```

### 历史消息 `listMessages`

按时间倒序分页，最新消息在前。

**请求**：
```json
{
  "chat_id": "1234567890",
  "limit": 20,
  "offset": 0
}
```

**响应**：
```json
{
  "uid": 1,
  "list": [
    {
      "msg_id": "9876543210",
      "chat_id": "1234567890",
      "from_uid": 1,
      "sender_name": "张三",
      "msg_type": "text",
      "content": "你好",
      "created_at": "2026-06-10T12:00:00Z"
    }
  ]
}
```

| 字段 | 说明 |
|---|---|
| uid | 当前用户uid，前端用于判断消息方向（自己/对方） |
| list | 消息列表，按时间倒序 |
| list[].sender_name | 发送者昵称，从用户表查询 |

### 清零未读 `clearUnread`

打开聊天详情时调用，将该会话的未读数清零。

**请求**：
```json
{
  "chat_id": "1234567890"
}
```

**响应**：`{}`

## 三、好友管理 `/long_chat/friend`

### 添加好友 `addFriend`

发起方自动通过，被添加方收到待确认请求。如果对方之前也加过你，自动互加好友。

**请求**：
```json
{
  "friend_uid": 2
}
```

**响应**：`{}`

### 同意好友 `acceptFriend`

**请求**：
```json
{
  "friend_uid": 1
}
```

**响应**：`{}`

### 拒绝好友 `rejectFriend`

**请求**：
```json
{
  "friend_uid": 1
}
```

**响应**：`{}`

### 删除好友 `deleteFriend`

双向删除，双方记录都会清除。

**请求**：
```json
{
  "friend_uid": 2
}
```

**响应**：`{}`

### 好友列表 `listFriends`

返回已通过的好友列表。

**请求**：无参数

**响应**：
```json
{
  "list": [
    {
      "uid": 2,
      "friend_uid": 1,
      "name": "张三",
      "status": 2,
      "created_at": "2026-06-10 12:00:00"
    }
  ]
}
```

### 待确认好友请求 `listPendingRequests`

**请求**：无参数

**响应**：
```json
{
  "list": [
    {
      "uid": 3,
      "friend_uid": 1,
      "name": "李四",
      "status": 1,
      "created_at": "2026-06-10 12:00:00"
    }
  ]
}
```

### 搜索用户 `search`

按账号、昵称或手机号模糊搜索，返回匹配用户列表及是否已是好友。

**请求**：
```json
{
  "keyword": "张"
}
```

**响应**：
```json
{
  "list": [
    {
      "uid": 2,
      "account": "zhangsan",
      "name": "张三",
      "is_friend": true
    },
    {
      "uid": 5,
      "account": "zhangwei",
      "name": "张伟",
      "is_friend": false
    }
  ]
}
```

## 四、成员管理 `/long_chat/member`

### 加入聊天 `joinChat`

将用户拉入群聊，仅群主可操作。

**请求**：
```json
{
  "chat_id": "1234567890",
  "uid": 5
}
```

**响应**：`{}`

### 退出聊天 `leaveChat`

退出群聊，群主不可退出（需先转让或解散）。单聊不可退出。

**请求**：
```json
{
  "chat_id": "1234567890"
}
```

**响应**：`{}`

### 踢人 `kickMember`

仅群主可操作。

**请求**：
```json
{
  "chat_id": "1234567890",
  "target_uid": 3
}
```

**响应**：`{}`

### 成员列表 `listMembers`

**请求**：
```json
{
  "chat_id": "1234567890"
}
```

**响应**：
```json
{
  "list": [
    {
      "chat_id": "1234567890",
      "uid": 1,
      "joined_at": "2026-06-10T12:00:00Z"
    },
    {
      "chat_id": "1234567890",
      "uid": 2,
      "joined_at": "2026-06-10T12:00:00Z"

```

## 五、WebSocket 推送消息格式

前端建立 WebSocket 连接后，收到的新消息推送格式：

```json
{
  "type": "long_chat",
  "data": {
    "chat_id": "123",
    "msg_id": "456",
    "from_uid": 1,
    "msg_type": "text",
    "content": "你好",
    "created_at": "2026-06-10T12:00:00Z"
  }
}
```

前端处理（使用 ws.js 组件）：
```javascript
import { ws } from '/static/_common/ws.js';

ws.on('long_chat', (data) => {
    // data.chat_id    - 聊天ID
    // data.msg_id     - 消息ID
    // data.from_uid   - 发送者uid
    // data.msg_type   - 消息类型
    // data.content    - 消息内容
    // data.created_at - 发送时间
});
```
